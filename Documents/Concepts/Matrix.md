# Matrix (행렬) — Unity 3D 좌표 변환의 기초

## Matrix란?

숫자를 격자(표) 형태로 배열한 것. 3D 그래픽에서는 **"위치, 회전, 크기를 하나의 표로 압축한 변환 도구"**이다.

```
오브젝트의 Transform 정보:
  Position = (10, 0, 5)
  Rotation = 45도
  Scale    = (2, 2, 2)

→ 이 세 가지를 하나의 행렬로 합칠 수 있다
```

---

## 4x4 행렬 — 왜 4x4인가?

3D 공간은 x, y, z **3개 축**이지만, 행렬은 **4x4**(16칸)를 쓴다.

```
┌                          ┐
│  회전+스케일  회전+스케일  회전+스케일  이동X │
│  회전+스케일  회전+스케일  회전+스케일  이동Y │
│  회전+스케일  회전+스케일  회전+스케일  이동Z │
│      0           0           0        1   │
└                          ┘
   ← 3x3 영역: 회전+스케일 →   ← 4번째 열: 이동 →
```

- **왼쪽 3x3 영역** — 회전과 스케일을 담당
- **4번째 열** — 이동(translation)을 담당
- **마지막 행 (0, 0, 0, 1)** — 원근 투영용 (일반 3D 오브젝트에서는 항상 고정값)

3x3만으로는 회전과 스케일밖에 표현 못 한다. **이동까지 하나의 행렬로 합치려고** 차원을 하나 늘려서 4x4를 쓴다.

## 4x4 행렬의 배치 규칙

```
┌ m00  m01  m02  m03 ┐
│ m10  m11  m12  m13 │
│ m20  m21  m22  m23 │
│ m30  m31  m32  m33 │
└                    ┘
```

각 자리의 역할:

```
┌ X축.x  Y축.x  Z축.x  이동.x ┐
│ X축.y  Y축.y  Z축.y  이동.y │
│ X축.z  Y축.z  Z축.z  이동.z │
│   0      0      0      1    │
└                              ┘
  1열     2열     3열    4열
  right   up     forward  position
```

### 각 열의 의미

**열(column) 단위**로 읽으면 의미가 보인다:

| 열 | 담는 것 | Unity에서 | 의미 |
|---|---|---|---|
| 1열 (m00, m10, m20) | X축 방향 | `transform.right` | "오른쪽이 어디인지" |
| 2열 (m01, m11, m21) | Y축 방향 | `transform.up` | "위쪽이 어디인지" |
| 3열 (m02, m12, m22) | Z축 방향 | `transform.forward` | "앞쪽이 어디인지" |
| 4열 (m03, m13, m23) | 이동 | `transform.position` | "어디에 있는지" |
| 4행 (m30, m31, m32, m33) | 고정 | (0, 0, 0, 1) | 원근용 (보통 안 건드림) |

### 규칙 요약

```
열을 읽으면:  각 축이 월드에서 어디를 가리키는지 (방향 + 크기)
4번째 열:     오브젝트가 월드 어디에 있는지 (위치)
축 벡터 길이: 그 축의 스케일
축 벡터 방향: 그 축의 회전 결과
```

Unity Inspector에서 보는 Position, Rotation, Scale이 전부 이 16개 숫자 안에 압축되어 있다.

## 상태별 행렬 예시

### 기본 상태 (위치만 있음, 회전/스케일 없음)

위치 `(10, 0, 5)`:

```
┌ 1  0  0  10 ┐     X축 = (1,0,0) → 오른쪽이 월드 오른쪽
│ 0  1  0   0 │     Y축 = (0,1,0) → 위쪽이 월드 위쪽
│ 0  0  1   5 │     Z축 = (0,0,1) → 앞쪽이 월드 앞쪽
│ 0  0  0   1 │     위치 = (10, 0, 5)
└              ┘
→ 단위 행렬(identity) + 이동만 있는 상태
```

### 스케일 적용

X축 2배, Y축 3배:

```
┌ 2  0  0  10 ┐     X축 = (2,0,0) → 길이가 2 = X방향 2배
│ 0  3  0   0 │     Y축 = (0,3,0) → 길이가 3 = Y방향 3배
│ 0  0  1   5 │     Z축 = (0,0,1) → 길이가 1 = Z방향 그대로
│ 0  0  0   1 │
└              ┘
→ 각 축 벡터의 길이 = 그 축의 스케일
```

### 회전 적용

Y축 기준 90도 회전:

```
┌ 0  0  1  10 ┐     X축 = (0,0,1) → 오른쪽이 월드 앞쪽을 가리킴
│ 0  1  0   0 │     Y축 = (0,1,0) → 위쪽은 그대로
│ -1 0  0   5 │     Z축 = (-1,0,0) → 앞쪽이 월드 왼쪽을 가리킴
│ 0  0  0   1 │
└              ┘
→ 축 벡터의 방향이 바뀜 = 회전
→ 축 벡터의 길이는 1 = 스케일 변화 없음
```

### 회전 + 스케일 동시

Y축 90도 회전 + X축 2배:

```
┌ 0  0  1  10 ┐     X축 = (0,0,2) → 방향: 월드 앞쪽, 길이: 2 (스케일 2배)
│ 0  1  0   0 │     Y축 = (0,1,0) → 방향: 월드 위쪽, 길이: 1
│ -2 0  0   5 │     Z축 = (-1,0,0) → 방향: 월드 왼쪽, 길이: 1
│ 0  0  0   1 │
└              ┘
→ 축 벡터의 방향 = 회전
→ 축 벡터의 길이 = 스케일
→ 둘이 자연스럽게 합쳐짐
```

## 스케일 행렬 만들기

대각선에 각 축의 배율을 넣으면 된다:

```
┌ 2  0  0 ┐     ┌ x ┐     ┌ 2x ┐
│ 0  3  0 │  ×  │ y │  =  │ 3y │
└ 0  0  1 ┘     └ z ┘     └  z ┘

점 (1, 1, 1)을 넣으면 (2, 3, 1)이 나온다 → 각 축 방향으로 늘어남
```

---

## 회전 행렬 만들기

삼각함수(sin, cos)를 사용한다. Y축 기준 45도 회전 예시:

```
┌ cos45°   0   sin45° ┐     ┌ 0.707   0   0.707 ┐
│   0      1     0    │  =  │   0     1     0    │
└ -sin45°  0   cos45° ┘     └ -0.707  0   0.707  ┘
```

점 `(1, 0, 0)`을 넣으면:

```
┌ 0.707   0   0.707 ┐     ┌ 1 ┐     ┌ 0.707 ┐
│   0     1     0    │  ×  │ 0 │  =  │   0   │
└ -0.707  0   0.707  ┘     └ 0 ┘     └ -0.707┘

→ 45도 돌아간 위치가 나옴
```

행렬의 장점은 **곱해서 합칠 수 있다**는 것:

```
최종 3x3 = 회전 행렬 × 스케일 행렬 → "늘린 다음 돌린다"가 하나의 행렬
```

## 3x4 vs 4x4

```
4x4 행렬:
┌ a  b  c  d ┐
│ e  f  g  h │    ← 4행 × 4열 = 16칸
│ i  j  k  l │
│ 0  0  0  1 │    ← 마지막 행은 거의 항상 (0,0,0,1) 고정
└             ┘

3x4 행렬:
┌ a  b  c  d ┐
│ e  f  g  h │    ← 3행 × 4열 = 12칸 (마지막 행 생략)
│ i  j  k  l │
```

마지막 행 `(0, 0, 0, 1)`은 **원근 투영**(카메라가 먼 물체를 작게 보이게 하는 것)에만 필요하다. 일반 오브젝트 좌표 변환에는 쓸 일이 없으므로 생략하면 **계산을 4번 덜 하니까 더 빠르다**.

## Unity Matrix4x4 메서드 비교

| 메서드 | 사용 행렬 | 하는 일 | 용도 |
|---|---|---|---|
| `MultiplyPoint(v)` | 4x4 전체 | 이동+회전+스케일+원근 | 카메라 투영 등 (느림) |
| `MultiplyPoint3x4(v)` | 3x4 (마지막 행 무시) | 이동+회전+스케일 | 일반 좌표 변환 (빠름) |
| `MultiplyVector(v)` | 3x3 (왼쪽 상단만) | 회전+스케일만 | 방향 변환 (이동 무시) |

### 사용 예시 — 메시 슬라이싱에서

```csharp
// 위치 변환: 월드 점 → 로컬 점
// 이동+회전+스케일 모두 필요, 원근은 불필요 → 3x4
Vector3 localPosition = source.worldToLocal.MultiplyPoint3x4(slicePosition);

// 방향 변환: 월드 방향 → 로컬 방향
// 이동은 의미 없음 (방향이니까) → Vector (3x3만 사용)
Vector3 localNormal = source.worldToLocal.MultiplyVector(sliceNormal).normalized;
```

### 왜 로컬 좌표로 변환해야 하나?

메시의 정점 데이터는 전부 로컬 좌표로 저장되어 있다. 절단 평면도 같은 좌표계로 맞춰야 비교가 가능하다.

```
월드 좌표: 적이 (10, 0, 5)에 서 있음
메시 내부: 정점은 여전히 (-0.5 ~ 0.5) 범위

절단 위치를 월드 좌표 그대로 (10, 0, 5) 넘기면
→ 메시 입장에서는 한참 바깥이라 아무것도 안 잘림

로컬로 변환해서 (0, 0, 0) 근처로 넘기면
→ 메시 정점들과 같은 좌표계이므로 정확히 잘림
```
