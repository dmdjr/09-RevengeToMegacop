# CLAUDE.md

이 파일은 Claude Code가 이 저장소에서 작업할 때 따라야 할 지침을 제공한다.

## 프로젝트 개요

**RevengeToMegacop** — Unity 3D 탑다운 액션 게임. 플레이어가 총, 투척 검, 텔레포트 수리검을 사용해 적 웨이브와 싸운다. 총알 패리/가드, 약화된 적 처형 가능.

- Unity 버전: 6000.0.67f1 (Unity 6)
- 언어: C# (글로벌 네임스페이스, 커스텀 네임스페이스 미사용)

## 운영 원칙

- 가정이 아닌 증거를 기반으로 한다. 결과를 최종 확인하기 전에 완료 주장을 하지 않는다.
- 품질을 유지하는 가장 가벼운 경로를 선택한다.
- SDK/프레임워크/API를 구현하기 전에 공식 문서를 먼저 확인한다.

## 위임 규칙

다음 작업은 전문 에이전트에 위임한다:
- 멀티파일 변경, 리팩토링, 디버깅, 코드 리뷰, 계획, 리서치, 검증

다음 작업은 직접 처리한다:
- 사소한 작업, 간단한 clarification, 단일 명령

## 에이전트 카탈로그

- `architect` (opus, read-only): 시스템 아키텍처 분석, 복잡한 버그의 근본 원인 추적. file:line 근거 필수
- `debugger` (sonnet): NullRef, 빌드 에러, 런타임 버그 추적. 최소 수정만
- `critic` (opus, read-only): 계획/코드 품질 게이트. REJECT/REVISE/ACCEPT 판정
- `executor` (sonnet): 명세 기반 구현. 최소 diff, 탐색 먼저
- `unity-code-architecture-reviewer` (sonnet): Unity C# 아키텍처 전문 리뷰

## 실행 프로토콜

- 광범위한 요청: 먼저 탐색한 후 계획한다
- 독립적인 작업 2개 이상: 병렬 실행
- 긴 작업(빌드/테스트): run_in_background 사용
- 작성과 리뷰는 별도 패스로 분리한다. 자기 작업을 자기가 승인하지 않는다
- 완료 전 확인: 미완성 작업 없음, 컴파일 에러 없음, 검증 증거 수집됨

## 검증 규칙

완료 선언 전에 반드시 검증한다. 검증 실패 시 반복한다.
- Unity C# 변경: 컴파일 에러 없음 확인
- 새 기능: 에디터에서 동작 확인
- 버그 수정: 재현 불가능 확인
